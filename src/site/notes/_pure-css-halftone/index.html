It should be easy to layer in additional colors with the same principle. Actually, let‚Äôs start with a simpler image. A gradient, to illustrate how halftone works in the first place. A [halftone](https://en.wikipedia.org/wiki/Halftone) pattern is an array of ink dots simulating the appearance of smooth gradiation of tones using just two pure tones (pure black ink and pure white background in this case). By varying the size of the dots, the average ink coverage in a given area simulates lighter or darker tones in that area. Dots large enough would bleed into each other, creating the effect of negative dots. Dot size and bleed can be emulated in one go using two simple image processing operations, **screen** and **threshold**. The first step is to *screen* the source image (in this case, the gradient) with a *blurry halftone pattern*. **Screen** is an operation that mixes the pixels of the source image and the overlay image using some kind of an inverted multiplication formula. Essentially, it *lightens lighter areas multiplicatively*. Because the dots are blurry, the screen operation gives us smaller dots in lighter areas on the original image and bigger dots in darker areas. Another way to put it is, lighter images chip more away from dots‚Äô fringes.
This operation is done via CSS `mix-blend-mode: screen`. The blurry dot pattern is generated using a `radial-gradient` as a repeated `background-image`, like this: The next step is to **threshold** the resulting image. That is, convert the image into pure black & pure white pixels. Dark pixels become fully black, and light pixels become white — according to some defined threshold between light vs dark. This creates the signature black-ink-matrix-on-white-paper look.
In CSS, there is no threshold filter, but it can be simulated by applying an extremely high **contrast** filter, pushing pixel values to the extremes of pure white and pure black. Effectively the same result as thresholding. In code, that‚Äôs simply a `filter: contrast(999)`. Another thing we can add is a **blur** filter, *just before* the thresholding operation. This emulates surface tension of the ink, or something.

Just add a set of dots for each of CMY — cyan, magenta, and yellow, the ‚Äúprimary colours‚Äù of ink — to complete the [CMYK](https://en.wikipedia.org/wiki/CMYK_color_model)! Make sure to stagger the dots so they are distributed evenly. How to stagger them well is left as an exercise to the dear reader, you (see [halftone angles](https://the-print-guide.blogspot.com/2009/05/halftone-screen-angles.html), [moir√© patterns](https://proofing.de/whats-moire-and-can-i-see-moire-in-a-proof/), etc). These will work just as well as black because the `contrast` filter operates on each RGB channel independently. The colours of cyan (#0ff), magenta (#f0f), and yellow (#ff0) are at their own extremes in each RGB channel, just like black (#000) and white (#fff). Thus, the `contrast` filter produces a similar thresholding effect on each colour independently and simultaneously! This is not a very accurate representation of halftone, mainly due to the operations being in RGB, not CMY. An accurate simulation would be to apply thresholding to each channel in some CMY space via JS or maybe WebGL. But it looks good enough in many cases.
